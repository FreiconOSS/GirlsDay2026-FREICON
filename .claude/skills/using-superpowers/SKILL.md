# 🦸 Using Superpowers — GirlsDay 2026

Du hast spezialisierte Skills für diesen Workshop. **Prüfe vor jeder Aktion ob ein Skill passt.**

## Verfügbare Skills

### Kern-Skills (Workshop-Flow)

| Skill | Wann |
|---|---|
| `brainstorming` | Jemand hat eine App-Idee → ZUERST dieser Skill |
| `vibe-coder` | Code wird geschrieben oder geändert → dieser Skill |
| `deploy-to-pages` | "Wir wollen die App online stellen" → dieser Skill |

### Unterstützungs-Skills (automatisch mitlaufen)

| Skill | Wann |
|---|---|
| `game-dev` | Die Mädels wollen ein Spiel bauen → Spielmechaniken, Engine-Wahl, Game-Loop |
| `asset-helfer` | App braucht Grafiken, Farben, Schriften → Emoji-Sprites, Paletten, Fonts |
| `code-reviewer` | Nach jedem größeren Feature → Code aufräumen, Fehler vermeiden |
| `debugger` | Etwas funktioniert nicht → Käferjagd-Modus! 🐛 |
| `making-of` | Nach jedem Feature-Schritt → Making-Of Seite automatisch aktualisieren |
| `weitermachen` | "Wie kann ich zuhause weitermachen?" → Erklärseite für nach dem Workshop |

## Die Regel

Auch wenn es schnell geht: **erst den passenden Skill laden, dann handeln.**

- Neue App-Idee → `brainstorming` (kein Code bevor der Plan steht!)
- Spiel bauen → `vibe-coder` + `game-dev` zusammen nutzen!
- Coding-Session → `vibe-coder` (nach jedem Feature auch `making-of` aktualisieren!)
- App soll schöner werden → `asset-helfer` für Farben, Emojis, Fonts
- Etwas ist kaputt → `debugger` (Ruhe bewahren, systematisch suchen)
- Code soll sauber sein → `code-reviewer` (leise im Hintergrund)
- "Wie mache ich zuhause weiter?" → `weitermachen`
- Deployment → `deploy-to-pages`

## Spezial: Voice-Modus

Wenn Eingaben kurz und gesprochener Sprache ähneln:
→ Antworte in 3-4 gesprochenen Sätzen, kein Markdown, kein Code in der Antwort.
→ Code kommt separat in die Datei.
