⚡ Schnell loslegen

🌐 Weg 1: KI im Browser (ohne Installation)

⭐ Super einfach — geht sofort

Du brauchst nur einen Browser und eine der beiden KI-Websites:

  1. Gehe auf chatgpt.com oder claude.ai (kostenlos!)
  2. Sag der KI zum Beispiel:
    „Ich möchte eine Browser-App bauen. Mach mir eine HTML-Datei mit einem Spiel wo man Punkte sammelt."
  3. Die KI schreibt den Code für dich
  4. Kopiere den Code in einen Texteditor (z.B. Notepad)
  5. Speichere die Datei als index.html
  6. Doppelklick auf die Datei — sie öffnet sich im Browser! 🎉
💡 Tipp: Je genauer du der KI sagst was du willst, desto besser wird das Ergebnis! Statt „mach ein Spiel" sag lieber „mach ein Jump-and-Run Spiel mit einer Katze die über Hindernisse springt".
📋 Eigene Kopie des Projekts erstellen

🍴 Was ist ein „Fork"?

⭐⭐ Ein bisschen Übung nötig

Ein Fork ist eine eigene Kopie unseres Projekts. Du bekommst alles — den ganzen Code, alle Bilder, alle Sounds. Und du kannst daran ändern was du willst, ohne dass unser Original kaputt geht!

Stell es dir so vor: Du kopierst ein Rezeptbuch und darfst in deiner Kopie eigene Rezepte ergänzen. 📖

Schritt 1: GitHub-Account erstellen

  1. Gehe auf github.com
  2. Klicke auf „Sign up"
  3. E-Mail eingeben, Passwort wählen, Username ausdenken
  4. Fertig! Du hast jetzt einen GitHub-Account! 🎉
💡 Kostenlos für Schüler! GitHub ist kostenlos. Es gibt sogar ein GitHub Student Pack mit Extras!

Schritt 2: Fork erstellen (im Browser)

  1. Gehe auf unser Projekt: github.com/FreiconOSS/GirlsDay2026-FREICON
  2. Klicke oben rechts auf den Button „Fork"
  3. Klicke auf „Create fork"
  4. Fertig! Du hast jetzt deine eigene Kopie! 🎉

🤖 Mit KI weiterarbeiten

Option A: VS Code + GitHub Copilot

⭐⭐ Empfohlen — so haben wir im Workshop gearbeitet!
  1. Lade VS Code herunter (kostenlos)
  2. Installiere die Erweiterung „GitHub Copilot" (im Extensions-Tab suchen)
  3. Melde dich mit deinem GitHub-Account an
  4. Öffne den Projektordner in VS Code: Datei → Ordner öffnen → GirlsDay2026-FREICON

  6. Sag z.B.: „Füge einen Countdown-Timer auf der Startseite hinzu"

📤 Änderungen hochladen

Deine Änderungen auf GitHub pushen

Wenn du etwas geändert hast und es online stellen willst:

Terminal
# 1. Alle Änderungen markieren $ git add -A # 2. Einen „Speicherpunkt" machen (wie Speichern im Spiel!) $ git commit -m "Mein cooles neues Feature" # 3. Auf GitHub hochladen $ git push # Fertig! Deine Änderungen sind jetzt auf GitHub! 🎉
💡 Wie Speichern im Spiel: Ein „Commit" ist wie ein Speicherpunkt. Du kannst immer zu jedem Speicherpunkt zurück! Mach also ruhig oft Commits.
🔀 Pull Requests — Änderungen vorschlagen

Was ist ein Pull Request?

Du hast deinen eigenen Fork (deine Kopie) und hast dort etwas Tolles gebaut. Jetzt willst du, dass deine Änderung auch im Original-Projekt landet — damit alle sie sehen können!

Dafür gibt es Pull Requests (kurz: PR). Das ist wie ein Vorschlag: Du sagst „Hey, schaut mal was ich gebaut habe! Wollt ihr das übernehmen?"

Das Original-Team (in unserem Fall Andreas) schaut sich deinen Vorschlag an und kann ihn annehmen. So arbeiten echte Programmierer auf der ganzen Welt zusammen! 🌍

So erstellst du einen Pull Request

  1. Pushe deine Änderungen auf deinen Fork (siehe oben)
  2. Gehe auf github.com zu deinem Fork
  3. Du siehst oben eine gelbe Box: „This branch is X commits ahead"
  4. Klicke auf „Contribute" → „Open pull request"
  5. Schreibe einen Titel, z.B.: „Neues Genre: Horror-Geschichten hinzugefügt 👻"
  6. Beschreibe kurz was du gemacht hast
  7. Klicke auf „Create pull request"
  8. Fertig! Jetzt wartet dein Vorschlag auf Review! 🎉
⚠️ Wichtig: Man kann nicht einfach so Code ins Original-Projekt hochladen — das wäre ja so als könnte jeder in deinem Zimmer die Möbel umstellen! Deshalb gibt es Pull Requests: Du fragst freundlich ob deine Änderung übernommen werden soll.

Tipps für gute Pull Requests

  • 📝 Guter Titel — sag kurz was du gemacht hast
  • 📸 Screenshot — zeig wie es aussieht (Bild in die Beschreibung ziehen)
  • 🧪 Testen — probier vorher aus ob alles funktioniert
  • 💬 Nett sein — Code-Review ist ein Gespräch, kein Test
  • 🎯 Klein halten — lieber 3 kleine PRs als eine riesige
